# OpenClaw 记忆分层与沉淀（可复用准则）

## 三层记忆

- **临时工作记忆（Working Buffer）**：会话草稿/中间态，必要时写入 `working-buffer.md`。
- **项目记忆（Project Memory）**：对某项目/系统长期有效的事实（端口、路径、服务名、配置、回滚/验收）。
- **长期原则/红线（Long-Term Rules）**：跨项目稳定有效的纪律补丁，收敛写入 `MEMORY.md`（索引/红线）。

## 触发：必须写入项目记忆的内容

- 新增或确认了 **稳定可复用** 的事实：端口、路径、环境变量、鉴权方式、常见报错与修复。
- 完成了一次 **可复现修复**：改了配置/脚本/服务单元，并通过验证。
- 发现了可复用的 **坑位规避**：端口冲突、路径不一致、权限、代理、模型协议差异等。

## 禁止写入长期记忆

- 未验证的猜测、一次性聊天内容、临时状态（例如“某进程刚好在跑”）。

## 推荐写法（便于检索）

每条记忆尽量包含：

- **结论一句话**
- **证据**（命令输出/日志路径/配置字段）
- **复现/修复步骤**（最短可执行）
